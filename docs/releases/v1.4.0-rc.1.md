# v1.4.0-rc.1

`v1.4.0-rc.1` 是 `v1.4.0` 的生产演练候选版本，不是最终稳定版本。

## 升级

- Action 运行时从 Ruby 迁移到 GitHub 原生 Node.js 24。
- 使用方应审核候选实现提交，并使用 README 中的完整提交 SHA；不要固定到可移动版本标签。

- 标签清单、策略文件格式、Action inputs 和 outputs 均无需迁移。
- 项目维护者可以在 `safety` 中显式配置删除行为和数量上限；省略该字段仍保持 v1.3 行为。

## 兼容性保证

- 默认仍为 dry-run。
- v1.3 的 inputs、outputs 名称和含义保持不变。
- 未配置 `safety` 时仍允许删除且不设置数量上限，与 v1.3 相同。
- 读取请求保持可重试；写入请求不会自动重试。
- 规划、部分失败计数、Unicode 处理和任务摘要继续由 Ruby v1.3 行为快照验证。

## 删除安全

- 所有目标仓库会在首个写请求前完成规划并汇总为不可变 `RunPlan`。
- `safety.deletions: deny` 禁止任何删除。
- `safety.max_deletions_per_repository` 限制单仓库计划删除量。
- `safety.max_deletions_total` 限制整次运行计划删除总量。
- 任一规则阻止执行时，写请求数为零；任务摘要会展示触发规则、删除总量和受影响仓库。
- MathArts 生产策略使用 `allow / 1 / 6`，依据是
  [2026-07-15 全组织预览](https://github.com/matharts/sync-labels-action/actions/runs/29416932472)
  覆盖 6 个合格仓库且计划删除为 0。

## 已知限制

- dry-run 不会被删除上限阻止；它会完整展示计划，维护者必须在写入前审查删除量。
- `repositories.exclude`、离线 `validate_only`、类型化错误和可归档计划文件不在本版本范围内。
- Node.js 24 真实写入演练完成前，不发布最终 `v1.4.0`。

## 验证

- `pnpm check`
- Node.js 24 持续集成
- 可复现 `dist/index.js` bundle 比对
- Actionlint workflow 校验
- 全组织 dry-run
