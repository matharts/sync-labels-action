# v1.5.0-rc.1

`v1.5.0-rc.1` 是 `v1.5.0` 的演练候选版本，不是最终稳定版本。

## 升级

- 新增可选的 `repositories.exclude` 策略；未配置时保持 `v1.4.0` 的仓库选择行为。
- 新增 `validate_only` Action input 和 `pnpm validate:config` 本地命令，用于不提供 GitHub 凭据、不访问网络的配置校验。
- 使用方应审核候选实现提交并固定完整提交 SHA；不要固定到可移动版本标签。

标签清单格式、现有 Action inputs 和 outputs 无需迁移。

## 仓库范围

- 选择顺序固定为：全部仓库或 `repositories.include` → `repositories.exclude` → 可选的 `repository` input。
- `repositories.exclude` 可以单独配置，也可以和 `include` 配合使用。
- 同一仓库不能同时出现在 `include` 与 `exclude` 中；名称比较忽略大小写，冲突会在访问 GitHub 前报错。
- 显式的 `include: []` 会报错，避免意外扩大同步范围；`exclude: []` 是合法的空排除规则。

## 离线校验

- `validate_only: "true"` 只读取 `config_file` 与 `policy_file`，不要求 `token` 或 `owner`，也不创建 GitHub 客户端。
- `pnpm validate:config` 使用与 Action 相同的 `GovernanceConfig` 规则；可通过 `--config-file` 和 `--policy-file` 指定文件。
- Action 与本地命令对非法配置返回一致的校验错误；非法配置使用非零退出码。

## 兼容性保证

- 默认仍为 dry-run。
- 未设置 `repositories.exclude` 和 `validate_only` 的 workflow 保持 `v1.4.0` 行为。
- `changed` 在预览模式表示完整计划是否包含变更；在写入模式表示是否实际完成至少一项变更。
- 删除安全、读取重试、写入不自动重试、部分失败计数和 Unicode 行为保持不变。

## 已知限制

- `validate_only` 不验证 GitHub 凭据、权限、仓库可见性或网络连通性。
- 本版本不引入稳定的错误原因分类、失败仓库输出或可归档计划文件。
- 并发、缓存和批处理优化不在本版本范围内。
- 仓库范围与离线校验演练完成前，不发布最终 `v1.5.0`。

## 验证门槛

- `pnpm check`
- Node.js 24 持续集成
- 可复现 `dist/index.js` bundle 比对
- Actionlint workflow 校验
- 包含 `exclude` 的全组织 dry-run
- 无 token、无 owner、断网环境中的 `validate_only` 校验
- 未设置新 input 的现有 workflow 兼容性演练
